# Deliv_arrangement — 全国 配車・積み合わせ最適化（VRP）ダッシュボード

東京都内（1日15件想定）を中心に、大阪・愛知・福岡・関東近郊6県（各1日3〜4件）・その他37道府県（各1日0〜1件）の出荷指示ダミーデータを月次で積み上げ、フィジカル制約（車格制限・付帯作業・時間指定・重量/容積ダブルチェック）を考慮した配車・積み合わせ最適化をブラウザ内でリアルタイムにシミュレートするダッシュボード。

- **構成**: 単一 `index.html`（Tailwind CDN + 標準SVG/HTML、外部チャートライブラリ不使用、モックデータ動的生成）
- **GitHub**: https://github.com/Logtk/delivery-consolidation-vrp
- **デモ**: https://logtk.github.io/delivery-consolidation-vrp/ （`Inv_mgmt_optim` と同方式。main直配信・Actions不要、pushで数分後に自動反映）
- **ローカル確認**: `index.html` をブラウザで開くだけ（Tailwind CDNのためネット接続が必要）

## 機能

1. **経営KPIカード（月間・全国）** — 総件数 / 総重量・容積 / 最適化前後の運賃 / 削減額・率（年間換算付き）、東京都内と地方拠点の内訳を併記
2. **表示日セレクタ（東京都内）** — 当月20営業日から1日を選択し、その日の東京都内出荷（詳細VRP対象）に絞って以下を表示
3. **車種別台数シフト比較グラフ** — 選択日の個別手配（ベタ付け）vs 最適化後（SVG棒グラフ、車種色は CVD 検証済みパレット）
4. **現場制約インサイトログ** — 車格制約による分割 / 越境混載による統合 / AM密度・付帯作業ペナルティをテキストで説明（選択日ベース）
5. **ルート別積載率詳細** — 車種・件数・重量/容積積載率（ボトルネック側を太字）・時間指定・越境フラグ・制約バッジ（選択日ベース）
6. **地方拠点サマリー** — 大阪・愛知・福岡・関東近郊6県・その他37道府県（集計）の月間件数・車両延べ台数・運賃を一覧表示
7. **Excelデータシミュレーター** — 月間全地域分（1,200件超）の元データを日付・都道府県フィルター＋ページネーションで閲覧
8. **動的パラメーター** — AM許容バッファ / 付帯作業ロスタイムのスライダーで全国データを即時再計算

## アルゴリズム概要（`index.html` 内 JS）

### 東京都内（詳細VRP、選択日単位）
- **積載判定**: 重量・容積(M3)・箱数のうち先に上限到達した側で足切り（車格別: 軽350kg/2M3/50箱、2t/4t/10t 同様）
- **パッキング**: ward-groupエリア別 First-Fit-Decreasing。2t限定先は上限2t車格で先行編成し、小口通常貨物を相乗り
- **車格選定**: ルート物量＋実効配送件数（付帯1件 = 1 + ロスタイム/10 件相当）を満たす最小車格。4t上限案と10t上限案の総運賃比較で安い方を採用
- **越境統合**: 積載率60%未満のルート同士を隣接エリア間でマージ（運賃が下がる場合のみ、越境フラグ付与）

### 地方拠点（大阪・愛知・福岡・関東近郊・その他道府県）
- 1地域・1日あたり0〜4件程度と小口のため、区部越境統合は行わず、車格制限のみを考慮した簡易パッキング（都度1〜2台便）で月間集計

### 共通
- **ベースライン**: 1件1台・最小適合車格で個別手配した場合の運賃と比較して削減効果を算出（月間・全国で積み上げ）
- **データ生成**: 当月の営業日20日分、地域ごとにポアソン分布（東京平均15件/日、主要都市・関東近郊平均3.5件/日、その他各県平均0.4件/日）で日次件数を生成

運賃モデル: 軽1.5万 / 2t 2.5万 / 4t 3.5万 / 10t 5.5万円（1台・1日あたり、架空値）
